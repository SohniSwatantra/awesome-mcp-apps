We love to see new MCP apps, servers, and tools added to the list. To ensure high quality, please follow these guidelines.

## How to Contribute

1.  **Fork the repository** on GitHub.
2.  **Clone your fork** locally.
3.  **Create a new branch** for your addition (`git checkout -b add-my-awesome-tool`).
4.  **Edit `README.md`** to add your entry in the appropriate category.
    *   Maintain alphabetical order within categories if possible.
5.  **Commit your changes** with a clear message (`git commit -m "Add My Awesome Tool"`).
6.  **Push to your fork** (`git push origin add-my-awesome-tool`).
7.  **Submit a Pull Request** against the `main` branch.

## Submission Guidelines

### General Requirements

*   **Relevance**: The item must be directly related to the **Model Context Protocol (MCP) APPS**.
*   **Quality**: Please only submit projects that are useful, documented, and working.
*   **No Spam**: Do not add entries purely for SEO or self-promotion without value.

### Link Formatting

Please follow this format for list entries:

```markdown
- [Name](Link) - Short, clear description.
```

*   **Name**: The name of the project or resource.
*   **Link**: Direct link to the project/resource (GitHub, Website, etc.).
*   **Description**: A concise explanation (ends with a period). limit to 1-2 sentences.

### Categories

Choose the most specific category for your entry:

*   **Featured MCPApps**: Production-ready applications.
*   **MCPApps (UI/Components)**: Frontend libraries, UI kits, and components (formerly MCP-UI).
*   **MCP Servers**: Implementations of MCP servers.
*   **Tools & SDKs**: Development tools and language bindings.
*   **Templates & Starters**: Boilerplate code.
*   **Tutorials & Learning**: Educational content.

### Screenshot Rules

If you are suggesting a featured app that benefits from a visual preview:
*   Screenshots are not currently hosted in this repo to keep it lightweight.
*   Link to a screenshot or video in your project's own README/Website instead.

## Pull Request Guidelines

*   Please ensure your PR description clearly describes what you are adding.
*   Check for duplicates before submitting.
*   If you are unsure where something belongs, ask in the PR or open an issue first.
